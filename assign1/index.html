    Make sure you have read Paul Ford's article "On HTML5 and the Group That Rules the Web" (see Reader on BlackBoard). What is, according to this article, the key difference between the HTML5 Recommendation published by W3C and the one published by the WHATWG?

    The W3C will continue to work on the HTML5 specification. WHATWG on the other hand will continue its work with HTML5 as a "Living Standard". The idea of a living standard is that it never settles down and is always being added to and refined.

    WHATWG sees the Web as an evolving, living platform; the W3C exists to formally recommend certain things. In practice, this means that it has been taking WHATWG its documents and changing them in ways that the WHATWG people do not always like.

  5. Question 5

    Summarize the key multilingual issues that W3C is addressing within the "Internationalization" activity. (Expected length: half page)

    The word 'internationalization' means that everyone in this world should be able to access the internet with no constrains. This also means that the developer has to create a page where there are no writing boundaries: text from left to right, from right to left or mixed. Other languages spoken by small countries might be taken into consideration, because they can add new writing rules which should be tackled and not ignored. It’s the developer responsibility to make sure that all the W3C standards are respected, to make sure that the final web page is easily adapted by every user from a different country, culture or language. This is where the Unicode character set from W3C comes in handy, because it allows to a seamless integration of human language into Web technologies.
