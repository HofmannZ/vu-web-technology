Introduction

This web page was built for the Web-technology course and we tried to implement the main guidelines, so the developers like us do not have to thing about the edge cases of usability and accessibility.

We put a lot of effort into keeping the page simple and easy to understand, therefore making it responsive and mobile compatible was not an issue. Our website can we easily scaled up and improved in terms of design and content because of the usage of descriptive tags: section and article, and class names.

Accessibility highlights

Guideline 1.3 'Adaptable' and Guideline 2.4 'Navigable': I think this is one of the most important guideline respected by our website, because we have a simple structure with a eye pleasant colors.
Making it simple allow the user to focus on the content and not on the process of making the content appear through some custom actions

Improvement plan

Considering the feedback from our TA, we could have added the sorted table for our page and improved the responsiveness of the web page.

Appendix A

Guideline Sub-guideline Explanation
1.1 Text Alternatives 1.1.1 Non-text Content

'Controls, Input': We don't have media controls, but for each input we have a type: ('text', 'data', 'number') and a descriptive ID or class.

Time-Based Media': We don't have such thing in our website.

'Test': We don't have such thing in our website.

'Sensory': We don't have such thing in our website.

'CAPTCHA': We don't have such thing in our website.

'Decoration, Formatting, Invisible': The website is structured exactly like a page with sections and articles, therefore removing the style will not have an effect on the content.
1.3 Adaptable 1.3.1 Info and Relationships You can programmatically determine the information, structure, and relationships, because the content of the page is wrapped according to it's purpose.
1.3.2 Meaningful Sequence Because the content of the page is ordered chronologically, a correct reading sequence can be programmatically determined.
1.3.3 Sensory Characteristics All our operating content is both visual and programmatically placed within the right context of the element.
1.4 Distinguishable 1.4.1 Use of Color Every element or action of the page are accessible even if the every color of the page is removed.
1.4.2 Audio Control We do not have any audio content in our website.
2.1 Keyboard Accessible 2.1.1 Keyboard We do not overwrite the main functionality of the keyboard, therefore the all functionality of the content is operable through a keyboard interface without requiring specific user actions.
2.1.2 No Keyboard Trap Because we do not overwrite the main functionality of the keyboard, the focus can be moved away from the component only a keyboard interface.
2.4 Navigable 2.4.1 Bypass Blocks The website contains titles for each section and has the navigation menu, that redirects the user to the desired section of the page.
2.4.2 Page Titled The website contains the right title for each section and subpage.
2.4.3 Focus Order The navigation sequence does not affect the meaning and operability of the website.
2.4.4 Link Purpose Each link of the page that redirects the user has the right descriptive title.
